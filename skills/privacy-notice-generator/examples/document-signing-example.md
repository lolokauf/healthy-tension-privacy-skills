# Privacy Notice Generator — Document Signing Platform Example

A worked example demonstrating the privacy-notice-generator skill applied to a document signing application with high-PII processing, electronic signatures, and multi-party document workflows.

---

## Input Description

A Next.js + PostgreSQL document signing platform with:

- **User accounts:** Email/password registration, Google OAuth, profile with name, email, signature image
- **Document workflows:** Upload PDFs, add signers by email, send signature requests, track completion
- **Signatures:** Canvas-drawn or uploaded signature images, typed signatures, signing certificates
- **Recipients:** Non-registered users receive documents via email with tokenised signing links
- **Email:** Resend API for transactional emails (signing requests, reminders, completion notices)
- **Storage:** PostgreSQL (Prisma ORM) on Railway, document files on AWS S3 (us-east-1)
- **Analytics:** PostHog with EU region (`eu.i.posthog.com`), memory persistence (no cookies)
- **Auth:** NextAuth.js with email/password + Google OAuth providers
- **Subscriptions:** Stripe for paid plans (monthly/annual)
- **No cookie banner** — application uses memory-only analytics, session cookie only
- **Data export:** Users can download their documents but no dedicated DSAR/export endpoint
- **Deletion:** Users can delete individual documents; no account deletion endpoint

**Data inventory available:** `data_inventory.yaml` in project root (composable mode).
